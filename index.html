Quick Start

Get up and running with Waffles Firmware in under 5 minutes. This guide covers basic setup and your first consent-aware application.

// C Implementation - Basic Setup 
#include <waffles.h>

void setup() {
  // Initialize Waffles core
  waffles_init();
  
  // Request user consent for data access
  if (consent_request("sensor_data_read")) {
    Serial.println("Consent granted");
  } else {
    Serial.println("Consent denied");
  }
}

void loop() {
  // Your application logic
  waffles_process_events();
  delay(100);
}
// Assembly - Low-level consent check 
MOV R0, #CONSENT_FLAG
BL  request_consent
CMP R0, #1
BEQ consent_granted
B   consent_denied

consent_granted:
  ; Execute authorized code
  BL  read_sensor_data
  B   done

consent_denied:
  ; Handle denial gracefully
  BL  display_error
  
done:
  BX  LR
// Advanced - Time-limited consent 
#include <waffles.h>
#include <waffles_time.h>

// Request consent valid for 5 minutes
consent_token_t token = consent_request_timed(
  "network_access",
  CONSENT_DURATION_MINUTES(5)
);

if (consent_is_valid(token)) {
  // Perform network operations
  wifi_connect();
  http_post("/api/data", sensor_data);
  wifi_disconnect();
  
  // Revoke consent early (optional)
  consent_revoke(token);
}
// Multi-permission request 
const char* permissions[] = {
  "sensor_access",
  "network_access",
  "storage_write"
};

consent_token_t tokens[3];
bool all_granted = consent_request_multiple(
  permissions,
  tokens,
  3
);

if (all_granted) {
  // All permissions granted
  read_sensors();
  upload_data();
  save_locally();
}

Ethical Principles

Frequently Asked Questions

Is Waffles Firmware really free?
Yes, completely free. Waffles is open-source under the MIT license. You can use it commercially, modify it, and redistribute it without any fees or royalties.
How does consent-first architecture work?
Every sensitive operation requires explicit user approval before execution. The firmware maintains a permission ledger and validates consent tokens before allowing operations like network access, sensor reading, or data storage. Permissions can be time-limited and revoked at any time.
What about performance overhead?
The consent system adds minimal overhead‚Äîtypically less than 3% compared to bare metal. Most operations are zero-copy and the permission checks are highly optimized. You can disable features you don't need at compile time to further reduce footprint.
Can I use this in commercial products?
Absolutely. The MIT license permits commercial use without restrictions. Many companies already use Waffles in production devices. We only ask that you contribute improvements back to the community when possible.
How is security audited?
We conduct quarterly security audits with independent third-party firms. All findings are published transparently. We also run an active bug bounty program and have formal verification for critical components.
What platforms are supported?
Currently 15+ platforms including Arduino, ESP32, Raspberry Pi Pico, STM32, Nordic nRF52, and more. We're constantly adding new platform support based on community requests.
How do I contribute?
We welcome all contributions! Check our GitHub repository for open issues, read the contributing guidelines, and join our community Discord. Contributions can be code, documentation, testing, or even financial support.
Is there commercial support available?
Yes, we offer enterprise support contracts including priority bug fixes, custom feature development, on-site training, and architectural consulting. Contact us for pricing.
